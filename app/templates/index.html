Architecture Overview

The system uses a six-agent architecture where each agent handles a distinct phase:

  • Ingestion Agent — Loads and normalizes vulnerability records from feeds
  • Context Agent — Builds dependency graph and matches vulnerabilities to services
  • Risk Reasoning Agent — Computes business-aware composite risk scores
  • Planning Agent — Ranks upgrades and assigns maintenance windows
  • Governance Agent — Enforces approval policies and compliance constraints
  • Explanation Agent — Generates human-readable rationale for each action
Scoring Formula
final_score =
  0.30 × severity +
  0.25 × exploitability +
  0.20 × business_impact +
  0.15 × blast_radius +
  0.10 × exposure
  − penalties(complexity, maintenance)

All sub-scores normalized to 0-100. Penalties subtracted for rollback complexity and maintenance window distance.
